# TODO Rails

A full-stack web-based task management application built with Spring Boot, Thymeleaf, MySQL, and vanilla JavaScript. Users can register, log in, and manage their own personalized to-do lists with priorities, types, and due dates.

---

## Features

- **User Authentication** — register, login, logout with BCrypt-hashed passwords
- **Protected Routes** — dashboard and task operations require login (Spring Security)
- **Task Management** — add, edit, delete tasks
- **Task Attributes** — title, description, priority (LOW / MEDIUM / HIGH), type (PERSONAL / OFFICIAL / STUDY / HEALTH / OTHER), due date, completion status
- **Dashboard Stats** — total, pending, and completed task counts
- **Client-side Validation** — JavaScript validates forms before submission
- **Server-side Validation** — Java `@Valid` annotations as a safety net
- **Flash Messages** — feedback after every add / edit / delete action
- **Custom Error Pages** — styled 404, 403, and 500 pages
- **Responsive Design** — works on desktop and mobile

## Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/sumanthd032/todo-rails.git
cd todo-rails
```

### 2. Create the database

MySQL will auto-create the `todorails` database on first run because of:
```
createDatabaseIfNotExist=true
```
But you can also create it manually:
```sql
CREATE DATABASE todorails;
```

### 3. Configure database credentials

Open `src/main/resources/application.properties` and update:

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/todorails?createDatabaseIfNotExist=true
spring.datasource.username=root
spring.datasource.password=YOUR_PASSWORD_HERE
```

### 4. Run the application

```bash
mvn spring-boot:run
```

Or run `TodoRailsApplication.java` directly from IntelliJ IDEA.

### 5. Open in browser

```
http://localhost:8080
```

## How It Works

### Authentication flow

```
Register → BCrypt hashes password → saved to users table
Login    → Spring Security loads user → verifies BCrypt hash → session created
Logout   → session destroyed → redirect to /login
```

### Security rules

```
Public:    /  /register  /login  /css/**  /js/**
Protected: /dashboard  /tasks/**  (redirect to /login if not authenticated)
```

### Task ownership

Every task stores a `user_id` foreign key. Before any edit or delete, the server verifies:
```java
task.getUser().getUsername().equals(loggedInUser.getUsername())
```
If the check fails, the user is silently redirected to their dashboard. This prevents URL manipulation attacks.

### Database schema (auto-generated by Hibernate)

```
users                          tasks
┌──────────────────────┐       ┌────────────────────────────┐
│ id          BIGINT PK│       │ id          BIGINT PK      │
│ username    VARCHAR  │◄──┐   │ title       VARCHAR        │
│ email       VARCHAR  │   │   │ description VARCHAR(500)   │
│ password    VARCHAR  │   │   │ priority    VARCHAR (enum) │
└──────────────────────┘   │   │ task_type   VARCHAR (enum) │
                           │   │ due_date    DATE           │
                           │   │ completed   BIT            │
                           └───│ user_id     BIGINT FK      │
                               └────────────────────────────┘
```
